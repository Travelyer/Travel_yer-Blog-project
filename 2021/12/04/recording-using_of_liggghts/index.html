liggghts模拟滑坡小结

发表于2021-12-04|更新于2021-12-05

开始前小吐槽

用liggghts模拟滑坡是出乎我的意料的，之前并没有接触过相关的模拟内容，不过也算是一次尝试，虽然经历了不少挫折，遭受了心情的起起落落。无论怎样⑧，最后还是完成了。

研究过程

初见liggghts

最初从陈泽学长那边拿到树莓派让我搞这个，我还是一脸懵逼；我以为我是继续搞并行方面的设置，结果他老董和学长已经搞完（可惜啊接手已经是差不多结尾了，要做的模拟滑坡来测试树莓派并行的效率。

最初是去啃liggghts的手册，了解代码的意思。代码还好，但是设置里面各种文件搞得人晕头转向的，还把输出的VTK以为是读入的模型。liggghts使用者少，国内基本⑤国外少，连个能问问题的论坛都没有。嗯啃的一个星期后没有办法，去浪费了正在考研的学长的半个小时当面询问，才搞懂是咋回事。

盒内小模型模拟

搞懂了大概是个什么样子之后，试着模拟了一下学长的实例模型，是个长方体（大概2x2x0.3)粒子堆向一面滑坡的过程。

事情仿佛一切向好在发展，老董让我去他办公室商谈一下 “半天大概就可以搞好了” 。过去展示了一下学习成果，liggghts大概是怎么个模拟法和已用代码的意思。接下来就是建好滑坡面的模型，让用粒子模拟一下滑坡体滑坡，然后输出速度矢量图和力链图就OK了（看起来很简单是⑧）

这力链图有够掉san的

建会儿模

虽然半天之内没搞好，不过大概明确的之后的目标。速度矢量图的输出有鹏展来帮忙做（力链图好像还没有办法输出）

最初试着用AUTOCAD建模，不过这个建三维模型不太好用；然后换了SU随手建了个模真是巨丑无比的模型，各种意义上都远离美学定义上的好康。不过至少导入到模拟中能用，粒子被模型墙壁分成了两半。

之后尝试继续用SU建滑坡面的模型，要用SU不加插件这少得可怜的功能来实现，感觉不太现实。董老师要求真实1:1比例的滑坡面。（大概长这样，带颜色的滑坡体，不过目前只模拟绿色和蓝色的1号、2号)

SU建个1立方米内的模型都费劲，让我建2250+ x 2785+ x 800+ 的滑坡面，着实建到明年都建不完。好在在鹏展建议下换了rhino，导入廖学长给的数据点后嵌面就可以生成滑坡面（极其舒适)。

痛苦源于无知

虽然模型建的很完美，但是导入模拟的时候却发生了不好的事情~~~~

最初的想法是在坡面上层生成粒子，让其沉积在坡面上，同时额外建空气墙防止其滑坡下去，沉积完成后删除空气墙让粒子开始沉积。

BUT！ 在开始沉积的过程，这些粒子，他们，穿模了！！！

就像模型不存在一般。我以为是模型的问题，于是在rhino重新建了几个设置不同的模型，但是无一例外，都穿模了

于是怀疑是SU和rhino导出模型可能有设置上的不同。便把rhino的模型导入SU再导出，还是不行；更加灵异的是，导入之前可用的丑模型之后，再导出也竟然不可用了。难道导入之后模型会发生什么灵异的突变嘛

总之反复调试过多遍之后，才发现SU导入STL文件会默认为毫米级别，意味着导入后会缩小1000倍大小，小到读入后根本没法影响粒子了。不过，就算大小问题解决后，滑坡面的模型依旧穿模，多洗叠，为什么会这样。（一些心碎瞬间）

为了找出问题，试了各种办法，还去读了粒子的内容（虽然二进制看不懂，ASCII看得懂但是也没用）

最后开始发狂乱改，把摩擦力什么的都改了一下，发现粒子密度调小1000倍后，就没有完全穿模了。（但还是有部分穿模，甚至有粒子飞到天上去）

好像不是模型的问题，但模型的困扰浪费了我两周的时间，也算个问题⑧

正确的方法是宝贵的金钥匙

但是，粒子密度缩小3个数量级，肯定是不符合实际的，模拟出来的东西是没有实际意义的（可是我们不是要研究算力的嘛><）。不过至少知道问题不在于模型而是在于各种设置。

于是开始疯狂改各种设置，发现，生成的离模型近的粒子不会穿模。但liggghts是lammps的扩展，lammps用于研究原子尺度各种反应的，所以并⑤正好在模型上方多少多少米生成粒子的这种命令（可惜并没有并行效率如liggghts的滑坡软件）。像滑坡面这种复杂的曲面，用liggghts那点生成规则体的命令来弄生成粒子的区域，让其刚好模型上面那么几米，难度跟用SU建滑坡面模型差不多。

董老师给了我一个方法，先建一个平面，用单个粒子改各种设置来测试出影响因子。我觉得这方法虽然理论上可行，但是大量粒子和单个粒子可能情况会不同，所以其实并没有怎么用。

于是尝试了一下邪道的方法，比如将同样的模型导入几次，在模型上方几米处加上同样的模型让粒子先接触，把杨氏模量拉到最高。可惜这样做只能减少却不能完全防止穿模，而且也不符合实际。

大量的粒子模拟需要很多的时间，不知不觉，从开始接触已经过了快一个月了，我将平时空余时间都用于模拟（都好久没打游戏了 T T），但是进展如此之慢令我感到十分焦虑，开始对自己的能力产生怀疑。之后跟董老师交流后，董老师再次建议我用那个方法，于是我抱着一种死马当活马医的想法去尝试。

在尝试之后，我才发现我当初的愚蠢。单个粒子模拟的速度很快，我用几个小时就发现粒子穿模的真正原因是粒子速度过高，而且仅仅与粒子与模型相交面的法向速度有关。而滑坡面和粒子的属性影响这个穿模的速度的值。

在某次去办公室的路上，发现模拟的竟然是三峡的黄土坡。正好有个会关于这个，难怪能拿到三峡的数据。

最后一点小问题来自粗心的自己

虽然找到了原因，但是安排好了一切之后，万众期待（就我自己）的时刻，他又穿模了啊啊啊啊。

已经要开始自暴自弃了，再次用出了邪道的方法；还写了一个循环来检测粒子速度，将速度过高的粒子的速度减少，又或者直接挥动铁拳，删掉这些粒子。However，邪道的方法总是不允许的，不过发现了，粒子并不是慢慢加速到穿模速度的，而是速度突变。

之后第无数次到老董办公室干活，交流的时候董老师说，可能是粒子碰撞之间的动能量交换，导致的粒子速度突变，正常粒子不会达到穿模的速度，并问了我的粒子密度和半径设置。我的粒子设置并⑤问题，可惜这是我的记忆的，去检查粒子设置的时候才发现，我在之前的一次测试，把一种粒子的半径拉大到了其他粒子1000倍。嗯，碰撞的动能交换结果可想而知。

于是天才的想法（不就是笨方法嘛）在我脑中产生了。先不设置重力，给要沉积的粒子一个朝向滑坡面的速度，在快接近的时候减小速度以防止穿模。等沉积的差不多了，再将粒子的速度设置为0，添加重力开始滑坡。

最终模拟了4500个粒子的滑坡，耗时4小时左右，不过是单个树莓派模拟，还不知道并行之后效率有多少提升。

一些令人兴奋的瞬间

代码及解析

atom_style granular
atom_modify map array
boundary f f f
newton off

communicate single vel yes

units si

最初的设置， granular 和 si 表示粒子的设置是自定非粒子尺寸粒子

region domain block -150.0 2500.0 -200.0 3000.0 -100.0 1000.0 units box （生成名为domain的长方体区域）
create_box 2 domain （2种type，domain区域内）

neighbor 20 bin
neigh_modify delay 0

生成粒子活动的整体范围，创建可以生成粒子的box

#Material properties required for new pair styles

fix m1 all property/global youngsModulus peratomtype 1.e7 1.e8 （杨氏模量）
fix m2 all property/global poissonsRatio peratomtype 0.42 0.42（泊松比）
fix m3 all property/global coefficientRestitution peratomtypepair 2 0.7 0.7 0.7 0.7（摩擦系数）
fix m4 all property/global coefficientFriction peratomtypepair 2 0.5 0.5 0.5 0.5（弹性系数）
#fix m5 all property/global k_finnie peratomtypepair 2 1.0 1.0 1.0 1.0（用不着的应力设置）
#fix m6 all viscous 0.5（会导致粒子减速的阻尼，但是可能会让粒子静止所以注释了）
#fix m6 all property/global characteristicVelocity scalar 2.

设定两种type的各种属性，杨氏模量1.e7的为粒子，1.e8的为组成滑坡面的粒子，其他都相同

#New pair style
pair_style gran model hertz tangential history #Hertzian without cohesion
pair_coeff * *

一些粒子的设置

timestep 0.000025

每个时间步相当于 0.000025s

#fix gravi_landslide1 all gravity 9.81 vector 0 -0.3031 -0.9530
#fix gravi_landslide2 all gravity 9.81 vector -0.2302 -0.2302 -0.9455

注释掉的，最初设置的沉积用重力

#the chute
#fix zwalls1 all wall/gran model hertz tangential history primitive type 1 zplane 0.0
#fix zwalls2 all wall/gran model hertz tangential history primitive type 1 zplane 800.0
#fix xwalls1 all wall/gran model hertz tangential history primitive type 1 xplane 0.0
#fix xwalls2 all wall/gran model hertz tangential history primitive type 1 xplane 2263.0
#fix ywalls1 all wall/gran model hertz tangential history primitive type 1 yplane 0.0
#fix ywalls2 all wall/gran model hertz tangential history primitive type 1 yplane 2820

注释掉的，不需要的墙

fix cad0 all mesh/surface file mesh/Top.stl type 2

导入模型，命名为cad0

fix Top all wall/gran model hertz tangential history mesh n_meshes 5 meshes cad0 cad0 cad0 cad0 cad0

根据模型生成墙 n_meshes 5 表示生成5个墙 meshes 后表示生成墙用的模型 多个相同的模型可以增大穿模所需速度的最小值

#distributions for insertion
fix pts1 all particletemplate/sphere 15485863 atom_type 1 density constant 2643 radius constant 0.0014
fix pts2 all particletemplate/sphere 15485867 atom_type 1 density constant 2643 radius constant 0.002
fix pts3 all particletemplate/sphere 32452843 atom_type 1 density constant 2643 radius constant 0.003
fix pts4 all particletemplate/sphere 32452867 atom_type 1 density constant 2643 radius constant 0.0034
fix pts5 all particletemplate/sphere 49979687 atom_type 1 density constant 2643 radius constant 0.004

生成粒子的种类，5种粒子都为type 1 密度为常数2643， 半径不同 0.0014 到0.004 共5种。（最后就是忘记改pts5的半径，导致的最后的问题）

fix pdd1 all particledistribution/discrete/numberbased 49979693 5 pts1 0.2 pts2 0.2 pts3 0.2 pts4 0.2 pts5 0.2

将5种粒子归为一种生成方式，生成的各种粒子分别占0.2

#region and insertion

region bc1 block 910 1320 1840 2350 180.0 190.0 units box
region bc2 block 940 1200 1670 1920 310.0 320.0 units box
region bc3 block 1477 1930 1720 2050 180.0 190.0 units box
region bc4 block 1400 1477 1820 2120 180.0 190.0 units box
region bc5 block 1200 1280 1760 1840 280.0 290.0 units box
region bc6 block 1280 1600 1760 1880 290.0 300.0 units box

生成粒子的6个正方体区域

#particle insertion
fix ins1 all insert/pack seed 67867967 distributiontemplate pdd1 vel constant 0.0 0.0 0.0 insert_every once overlapcheck yes all_in yes particles_in_region 1600 region bc1

fix ins6 all insert/pack seed 67867967 distributiontemplate pdd1 vel constant 0.0 0.0 0.0 insert_every once overlapcheck yes all_in yes particles_in_region 300 region bc6

在不同区域一次性生成不同数目的粒子

#apply nve integration to all particles that are inserted as single particles
fix integr all nve/sphere

设置粒子更新方式

#output settings, include total thermal energy
compute 1 all erotate/sphere
thermo_style custom step atoms ke c_1 vol
thermo 1000
thermo_modify lost ignore norm no

模拟过程中输出的内容 step 时间步 atoms 粒子数 ke 动能 c_1 貌似是碰撞的摸着数据 vol 体积

#insert the first particles so that dump is not empty
run 1
dump dmp all custom/vtk 200 post/chute_*.vtk id type type x y z ix iy iz vx vy vz fx fy fz omegax omegay omegaz radius

运行1步，设置输出在文件内的粒子的内容

#run
unfix ins1

unfix ins4

unfix ins6

结束生成粒子过程，让生成的粒子可控

group atom1 region bc1
group atom2 region bc2
group atom3 region bc3
group atom4 region bc4
group atom5 region bc5
group atom6 region bc6

将不同区域内的粒子加到不同的组内

displace_atoms atom1 rotate 1115 2135 185.0 -1 0 0 14.45

displace_atoms atom6 rotate 1440 1680 295 0 0 -1 45
displace_atoms atom6 rotate 1440 1680 295 -1 0 0 19

翻转长方体的粒子堆，让其整体平行滑坡面

set group atom1 vx 0 vy -6.259 vz -24.20
set group atom2 vx 0 vy -14.694 vz -20.225
set group atom3 vx -5.755 vy -5.755 vz -23.6375
set group atom4 vx -5.755 vy -5.755 vz -23.6375
set group atom5 vx 0 vy -3.130 vz -12.10
set group atom6 vx -2.8875 vy -2.8875 vz -11.81875

给粒子垂直要沉积的面的初速度

run 85000
set group atom3 vx -0.5755 vy -0.5755 vz -2.36375
set group atom4 vx -0.5755 vy -0.5755 vz -2.36375
set group atom6 vx -0.5755 vy -0.5755 vz -2.36375
set group atom5 vx 0 vy -0.6259 vz -2.420

run 55000
set group atom1 vx 0 vy -.6259 vz -2.420
set group atom2 vx 0 vy -1.4694 vz -2.0225

在粒子接近滑坡面的时候更改粒子速度，以防穿模

run 110000
set region domain vx 0 vy 0 vz 0
unfix Top
fix Top_new all wall/gran model hertz tangential history mesh n_meshes 1 meshes cad0

进行最后的低速沉积后将所有粒子速度设置为0，将之前的5个模型的叠加墙删除，换作一个模型的正常墙。

fix graviy all gravity 9.81 vector 0.0 0.0 -1.0

run 1150000
